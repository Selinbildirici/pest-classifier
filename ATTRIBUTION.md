## AI Assistance

### Tool Used

**Claude (Anthropic)**, accessed through claude.ai across multiple
conversation sessions during project window.

### How Claude Was Used

Claude was used in an **iterative, agentic-mode-style workflow** in which
the model helped scaffold code, debug issues, and draft documentation.
All design decisions, run configurations, hyperparameter choices, and
final integration choices were authored by me. Claude's role can be
characterized in the following categories:

#### 1. Code Scaffolding (Substantial AI Assistance)

Initial scaffolding for the following components was generated by Claude
based on my specifications, then reviewed, modified, and integrated by me:

- **PyTorch training loop structure** (mixed precision, differential LR,
  cosine annealing scheduler, early stopping)
- **Albumentations augmentation pipeline class wrapper**
  (`AlbumentationsDataset`)
- **Per-class F1 / confusion matrix computation logic**
- **Grad-CAM `vit_reshape_transform` function** (the patch-token-to-2D
  reshape trick required for applying Grad-CAM to Vision Transformers)
- **Softmax averaging ensemble logic**
- **Helper functions for visualization** (matplotlib subplot grids,
  heatmap rendering, side-by-side gallery layout)

For each of the above, I specified the inputs/outputs, hyperparameters,
and integration requirements. Claude generated the boilerplate, which I
verified against documentation, debugged in Colab, and integrated into
the notebook. In several cases I rejected Claude's first attempt and
asked for revisions (e.g., the gallery cell went through 4 iterations
because the reference images were unrepresentative).

#### 2. Debugging and Iteration (Joint Work)

When training behaved unexpectedly or visualizations looked off, I
described the symptoms to Claude and we debugged together. Examples:

- **Confusion matrix axis interpretation:** I initially had rows and
  columns swapped in my interpretation; Claude helped clarify the
  sklearn convention.
- **ImageFolder vs. classes.txt index offset:** Discovered through
  exploration that ImageFolder's alphabetical sort produces a +1 offset
  relative to the IP102 classes.txt 1-indexed IDs. Claude helped me
  verify this hypothesis with a sanity-check cell.
- **Grad-CAM blank heatmaps initial issue:** Resolved by adding the ViT
  reshape transform (Claude pointed me to the standard pattern).
- **Side-by-side gallery refinement:** I rejected three of Claude's
  initial attempts because the reference images were unrepresentative.
  Final version uses manual `skip`
  parameters for problematic rows.

#### 3. Writeup Drafts (Iterative Collaboration)

For the README's "Approach", "Evaluation", "Error Analysis", "Grad-CAM
Analysis", and "Quantitative & Qualitative Evaluation" sections, I
worked iteratively with Claude to articulate the findings I had
generated. The actual results, observations, and analytical insights
came from me reading the outputs of my experiments. Claude helped me
phrase them clearly. Specifically:

- The **four failure-mode taxonomy** (fine-grained intra-genus,
  lifecycle-stage, contextual, class-label ambiguity) emerged from my
  reading of the confusion pair analysis output. I described the
  patterns I noticed.Claude helped me name and structure these into a
  coherent taxonomy.

Throughout this iteration, I rejected drafts that overclaimed findings and
insisted on Run 3 as the consistent reference model throughout the project. 
The finaltext reflects my readings of the data, written in my voice with
Claude's drafting assistance.

#### 4. Project Planning (AI-Assisted)

Claude assisted with day-by-day project planning: which experiments to
run, what order to run them, when to stop training, how to fit the work
within a timeline. The blueprint document
(`project_blueprint4_copy.pdf` referenced internally) was a result of
this planning conversation.

### What Was NOT AI-Generated

- **Actual model training:** All 8 training runs were executed by me on
  Google Colab Pro with my own data and compute resources.
- **All numerical results** (test accuracies, F1 scores, confusion matrix
  entries, ensemble gains) come from my actual runs.
- **Choice of which examples to include in galleries and Grad-CAM:**
  Made by me through iterative review (rejecting Claude's first picks
  multiple times).

I am fully responsible for the work submitted. Where Claude generated

## Dataset

**IP102: A Large-Scale Benchmark Dataset for Insect Pest Recognition**

> Wu, X., Zhan, C., Lai, Y. K., Cheng, M. M., & Yang, J. (2019). IP102:
> A Large-Scale Benchmark Dataset for Insect Pest Recognition. In
> *Proceedings of the IEEE/CVF Conference on Computer Vision and Pattern
> Recognition (CVPR)*, pp. 8787–8796.

The dataset is available for academic use. I accessed it via the
Kaggle mirror at `rtlmhjbn/ip02-dataset`.

## Pretrained Model Weights

All pretrained backbone weights are loaded from the **timm** library,
which sources them from publicly available checkpoints:

- **ViT-Base/16:** Pretrained on ImageNet-21k by Google Research,
  fine-tuned on ImageNet-1k. Released under Apache-2.0.
- **ResNet-50:** Pretrained on ImageNet-1k by torchvision, included in
  timm. Released under BSD-3.
- **EfficientNet-B0:** Pretrained on ImageNet-1k by Google, included in
  timm. Released under Apache-2.0.
